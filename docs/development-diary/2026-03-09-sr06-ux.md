# Diário de Desenvolvimento — 2026-03-09 (SR-06)

**Sprint:** SR-06 — UX para Produto Low-Ticket

---

## Contexto

O produto estava tecnicamente sólido, mas o onboarding era inacessível para o público-alvo: pequenos empreendedores sem conhecimento técnico (clínicas, restaurantes, prestadores de serviço).

O objetivo desta sprint foi transformar a experiência de configuração de "para desenvolvedores" para "qualquer pessoa consegue".

---

## Problemas Identificados

| Problema | Impacto |
|---|---|
| README focado em comandos de terminal | Assustador para não-técnicos |
| `.env.example` com jargão técnico sem contexto | Cliente não sabe o que preencher |
| Nenhum guia explicando como criar contas no Meta/Google | Principal ponto de abandono |
| Nenhum script interativo | Exigia conhecimento de terminal |
| Nenhum FAQ | Dúvidas frequentes sem resposta |

---

## Entregas da Sprint

### Novo: `docs/guia-passo-a-passo.md`
Guia completo de configuração em português simples, escrito para quem nunca mexeu com APIs. Cobre:
- Criação da conta no n8n (Cloud e local)
- Passo a passo completo de configuração da Meta API (App → WhatsApp → Webhook)
- Criação da planilha Google Sheets com Service Account
- Geração e importação do workflow
- Como testar o sistema
- Checklist de credenciais

### Novo: `docs/faq.md`
Perguntas frequentes com soluções práticas para os problemas mais comuns:
- Bot não responde
- Token expirado (com instrução para token permanente)
- Google Sheets não recebe dados
- Como alterar as perguntas do bot
- Segurança e credenciais expostas
- Limites de uso

### Novo: `scripts/configurar.sh`
Assistente interativo em português. O cliente executa `bash scripts/configurar.sh` e responde perguntas simples. O script:
- Coleta todas as informações necessárias
- Gera o arquivo `.env` automaticamente
- Instala dependências (`npm install`)
- Gera o workflow personalizado
- Faz o deploy para o n8n
- Exibe as instruções finais de ativação

### Atualizado: `README.md`
Reescrito com a seção "COMECE AQUI" no topo, direcionando para o guia passo a passo. Linguagem focada no benefício ("o bot responde automaticamente às 3 da manhã"), não em features técnicas.

### Atualizado: `.env.example`
Comentários reescritos em português simples, explicando onde encontrar cada informação e o que cada variável faz.

---

## Decisões de Design

**"Assistente interativo em vez de documentação só":** Não-técnicos não leem documentação extensa. Um script que faz perguntas é mais provável de ser concluído com sucesso do que um guia com 5 etapas.

**"Linguagem de benefício no README":** "O bot responde às 3 da manhã" é mais convincente que "respostas automáticas via WhatsApp Business API".

**"Checklist de credenciais":** O maior ponto de abandono é o cliente não saber o que precisa antes de começar. A checklist resolve isso.
