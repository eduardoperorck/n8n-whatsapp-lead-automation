## 2. Análise de Risco Pré-Deploy (Decisões de Resiliência)

Antes de escrever qualquer código, mapeei os pontos críticos de falha:

| Ponto de Falha | Risco | Estratégia Implementada |
|---|---|---|
| Webhook n8n | Payloads malformados / áudio 50MB | Parser defensivo com limite 64KB + rejeição por tipo |
| WhatsApp API | Rate limit (burst de mensagens) | Retry com backoff exponencial (1s, 2s, 4s) |
| Google Sheets API | Quota diária (100 req/100s) | Retry com jitter + fallback: log + notificação manual |
| Credenciais | Vazamento por hardcode | workflowValidator.js bloqueia qualquer secret literal |
| n8n reinício | Workflows perdidos | importWorkflow.sh idempotente (PUT se já existe) |
| Status updates do WhatsApp | Loops desnecessários | Parser ignora silenciosamente eventos de status |
| Sessão expirada | Estado corrompido | Timeout de 30min reseta para IDLE |

**Decisão crítica:** O webhook sempre retorna HTTP 200 — mesmo em erro interno. Isso é obrigatório pois o WhatsApp re-envia a mensagem para qualquer status != 200, criando loops infinitos.

## 3. Small Releases Executadas

### SR-01 — Fundação do Repositório ✅
**Arquivos criados:**
- `.gitignore`, `.env.example`, `README.md`
- `docs/architecture.md`, `docs/product-overview.md`, `docs/workflow-logic.md`
- `workflows/template/whatsapp_lead_template.json` — 16 nodes, pipeline completo
- `examples/form_payload.json`, `examples/lead_data.json`

**Decisão arquitetural:** O template usa `{{variavel}}` como strings literais dentro do JSON — o arquivo é JSON válido com placeholders que o templateEngine substitui por regex antes do parse.

**Nodes do template:**
1. WhatsApp Webhook (trigger com suporte GET/POST)
2. Method Router (IF: verifica GET vs POST)
3. Verify Challenge Response (handshake Meta)
4. Message Parser (Code: validação defensiva completa)
5. Message Valid? (IF: rota válidos vs inválidos)
6. Send Unsupported Type Fallback (resposta para áudio/imagem)
7. Intent Router (Code: máquina de estados)
8. Intent Switch (Switch: greeting/qualifying/complete/unknown)
9. Send Welcome Message
10. Lead Qualification (Code: Q&A progressivo 5 perguntas)
11. Send Next Question
12. Send Completion Message
13. Lead Storage — Google Sheets
14. Agent Notification
15. Send Unknown Intent Fallback
16. Respond 200 OK

### SR-02 — Workflow Generator ✅
**Arquivos criados:**
- `generator/templateEngine.js` — carregamento, substituição segura (escape de chars JSON), detecção de variáveis não resolvidas
- `generator/generateWorkflow.js` — orquestração completa, CLI, geração de ID único, remoção de campos internos, metadata de geração
