# CLAUDE.md

Project: n8n WhatsApp Lead Automation

This file defines the engineering rules and product direction for this repository.

Claude Code must follow these guidelines when generating or modifying code.

---

# Product Vision

Build a production-ready WhatsApp automation system using n8n.

The system must:

- automatically reply to incoming messages
- qualify leads using structured questions
- store leads in Google Sheets
- notify a human agent
- support automated workflow generation

The project also functions as:

- a commercial automation product
- a reusable automation template
- a technical portfolio project

---

# Architecture Principles

Claude must follow these principles:

1. Modular design
2. Workflow reusability
3. Clear separation between templates and generated workflows
4. Environment-based configuration
5. No hardcoded credentials
6. Agile Vibe Coding & Previsão Pós-Deploy: Desenhar o sistema considerando edge cases reais (timeout, falhas de rede, limites do WhatsApp).
7. Benchmark Driven: Provar conceitos complexos com scripts (PoC) simples antes de integrá-los no projeto principal.
8. Small Releases: O desenvolvimento deve ser iterativo, quebrando o trabalho em pequenos PRs ou commits perfeitamente funcionais ("one-shot prompt é mito").

---

# Repository Structure Rules

Claude must maintain the following structure:

workflows/
contains all n8n workflow files

generator/
contains workflow generation logic

docs/
contains architecture and product documentation
docs/development-diary/
contains chronological logs recording progress and architectural decisions

scripts/
contains deployment scripts

examples/
contains example payloads

---

# Workflow Engineering Rules

All workflows generated by Claude must:

- be compatible with n8n
- contain clear node naming
- include logical grouping of nodes
- support easy customization
- include error handling
- implement Circuit Breakers, fallbacks and retries where applicable for external APIs
- be developed through Small Releases: iterar em entregas parciais em vez de construir tudo de uma vez.

---

# Security Rules

Claude must never:

- commit credentials
- hardcode API keys
- expose tokens
- store secrets in workflow JSON

All secrets must be referenced via environment variables.

---

# Workflow Pattern

All WhatsApp workflows must follow this structure:

Webhook
    ↓
Message Parser
    ↓
Intent Router
    ↓
Lead Qualification
    ↓
Lead Storage
    ↓
Agent Notification
